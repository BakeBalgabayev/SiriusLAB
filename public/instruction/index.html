Синхронизация 1С с МС

Интеграция с 1С: Бухгалтерия версии 8.3 и выше для Казахстана

⚠️ Текущие ограничения

🚫 На данный момент решение не поддерживает базы, где вместо организаций используются структурные подразделения.

Если вы используете несколько договор эквайринга одного банка, то для проведения документа "Оплата покупателей платежными картами" в 1С необходимо вручную выбирать Договор эквайринга.

Если документы передаются из 1С в МС, где имеются обязательные дополнительные поля, то их необходимо убрать обязательность их заполнения

1С: Fresh не поддерживается - решение находится в разработке. Если вам нужна синхронизация, то рекомендуем перейти с 1С: Fresh на серверную версию.

Виды синхронизации:

- Отправка данных с сервиса МойСклад в 1С;

- Отправка данных с 1С в сервис МойСклад;

- Взаимный обмен данными.

- Запуск полного переноса данных в другую систему.

- Автоматический запуск синхронизации согласно расписанию.

Поддерживаемые документы:

- Заказ покупателя;

- Заказ поставщику;

- Приемка;

- Возврат поставщику;

- Оприходование;

- Перемещение;

- Отгрузка;

- Списание;

- Возврат покупателя;

- Входящий платеж;

- Исходящий платеж;

- Приходный кассовый ордер;

- Расходный кассовый ордер;

- Техоперации;

- Производственные задания.

Поддерживаемые справочники:

- Товары;

- Группы товаров;

- Контрагенты;

- Договоры;

- Комплекты;

- Техкарты.

Функции приложения:

- Выбор вида синхронизации;

- Выбор документов и справочников для обмена;

- Пошаговая первичная настройка обмена;

- Соответствие справочных данных;

- Настройка значений по умолчанию;

- Запуск частичного либо полного обмена;

- Запуск обмена с определенной даты;

- Отчет с настройками обмена;

- Автоматический запуск обмена (гибкие настройки периода).

Обмен документами и справочниками между «МоимСкладом» и «1С: Бухгалтерия» осуществляется с использованием JSON API.

Логика работы

Принцип «Источник — главный»

Документ из МС → необходимо редактировать только в МС.

Документ из 1С → необходимо редактировать только в 1С.

Проведение документов

По умолчанию отключена функция автоматического проведения загруженных документов.

При включённой опции "Автоматически проводить документы после загрузки" загружаемые документы из МС автоматически будут проводится в обоих системах.

⚠️ Включайте только по согласованию с бухгалтером, чтобы избежать перепроверок со стороны бухгалтера!

Единая нумерация из МС

Если в настройках обмена включено: "Отображать номер документа из МС в печатной форме ЭСФ"

тогда при обмене документом "Отгрузка" в печатных формах ЭСФ 1С (например, 3-2, Р-1) будет отображаться номер из МС в поле номер документа поставки.

Логика сопоставления товаров и контрагентов

📦 Товары (Номенклатура)

Алгоритм поиска при обмене:

по полю артикул;

по полю внешней код.

Если объект не найдён, то создается новый товар/услуга.

Группы и иерархия номенклатуры поддерживаются.

🧾 Контрагенты

Алгоритм поиска при обмене:

по полю ИНН в МС и ИИН/БИН в 1С;

по полю внешний код.

Рекомендуем сопоставить товары и контрагенты перед началом запуска синхронизации во избежание дублей.

Рекомендуем проводить работы в нерабочее время или когда вы уверены, что никто не использует 1C.

1.Настройки в МС

Синхронизация данных выполняется по основной (одной) организации, указанной в МоёмСкладе.

В случае необходимости настройки синхронизации по нескольким организациям необходимо обратиться в техническую поддержку.

Укажите ИИН/БИН компании

  1. В МоёСкладе перейдите в раздел Настройки пользователя.
  2. В разделе Значения по умолчанию, рядом с полем Организация нажмите на .
  3. Затем в разделе Юридические реквизиты проверьте/заполните поле ИИН и нажмите кнопку Сохранить. Для ИП укажите - ИИН, а для ЮЛ - БИН.
Если ИИН/БИН компании в 1С и МоёмСкладе не совпадет, обмен данными не будет работать корректно.

Настройки обмена в приложении МС

  1. Откройте приложение для выбора документов и справочников для обмена. Выберите способ обмена:
    • При нажатии на кнопку Загружать все в 1С все документы и справочники будут отправляться с сервиса МойСклад в 1С.
    • При нажатии на кнопку Выгружать все с 1С все документы и справочники будут отправляться с 1С в сервис МойСклад.
    • При нажатии на обе кнопки будет установлен взаимный обмен данными.
    В случае установления режима взаимного обмена, изменения ранее переданных документов и справочников не производятся. Данное ограничение будет устранено в следующих версиях приложения.
  2. Уберите отметку в документах и справочниках, которые не хотите синхронизировать, и нажмите кнопку Сохранить настройки

4.Запуск обмена

Имеется несколько способов начала обмена:

  • Начать обмен с определенной даты (рекомендуемый)
  • Запустить полный обмен вне зависимости от даты (запускайте только если одна из систем пустая!)

Установите дату запрета синхронизации. Документы до указанной даты не будут передаваться даже если будут внесены изменения.

Рекомендуем начинать первый запуск обмена в нерабочее время или когда вы уверены, что никто не использует 1C.

Последующие обмены можно запустить через кнопку Выполнить обмен или настроить автоматические запуски синхронизации.

При запуске обменов будет отображаться следущее окно.

После завершения обмена вам отобразится статус обмена.

5.Автоматический запуск синхронизации

Регулярный запуск при файловой базе 1С возможно только при наличии минимум 1го активного сеанса
  1. В меню программы выберите пункт Администрирование и подпункт Печатные формы, отчеты и обработки, затем выберите Дополнительные отчеты и обработки. Обратите внимание чтобы галочка Дополнительные отчеты и обработки была включена.
  2. Нажмите на кнопку Добавить из файла, затем отобразится окно с предупреждением, нажмите Продолжить
  3. Разархивируйте загруженный файл с плагином, затем в открывшемся окне выберите файл МСОбменРегЗадание.epf для продолжения установки.
  4. Нажмите на значение столбца Расписание.
  5. В открывшемся окне вы можете настроить автоматические запуски. Рекомендуем во вкладке Дневное установить значение Повторять через = 900 секундам (15 мин).
  6. Рекомендуем во вкладке Общее установить значение Повторять каждые = 1 дн. Соотвественно каждые 15 минут каждый день будет производиться запуск синхронизации.
  7. Нажмите Записать и закрыть
Автоматические запуски синхронизации будут производится только при условии запущенной базы 1C.

Возможные ошибки и их решения

Истекла подписка на решение

🔧 Решение: Подключите решение сразу на год — получите скидку 20% и сэкономьте от 84 000 тг. Забудьте про продление на целый год!

Ошибка при передаче Реализации / Отгрузки

💥 В логах: «Отгрузка невозможна. Недостаточно остатков для товара»

🔧 Решение: Проверьте актуальность остатков в 1С и МС. Перед синхронизацией оформите документ Оприходование.

Документы из 1С не передаются или перестали приходить в МойСклад

📄 Частая причина: Установлены обязательные поля в МС, которые 1С не может заполнить.

🔧 Решение: Уберите обязательность у этих полей — синхронизация восстановится.

Синхронизация полностью остановилась

🌐 Решение: Убедитесь, что сервер 1С подключён к интернету. Без связи — без синхронизации.

  • Будет реализована более точная обработка специфики производственного учёта и движение по техкартам.
  • Поддержка 1С:Fresh
  • Синхронизация документов из раздела “Розница”
  • Редиайн решения

🔖 Версия 1.1.0.8

📄 Выгрузка документов Списание и Перемещение из 1С в МойСклад

🔢 Единая нумерация с МойСклад
> Если включена настройка “Отображать номер документа из МС в печатной форме ЭСФ”, то номер из МС подставляется в печатные формы (напр., 3-2, P-1) и в ЭСФ
✅ Автоматическое проведение документов после загрузки
> Включайте 🟡 только по согласованию с бухгалтером, чтобы избежать ошибок в учёте!
🔁 Новая логика синхронизации документов
❌ Было: Взаимный обмен вызывал циклическую отправку при изменении одного и того же документа.
✅ Стало: Система определяет источник создания документа.
Документ из МС → редактируется только в МС
Документ из 1С → редактируется только в 1С
